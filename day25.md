# Day 25 – DevOps Monitoring Practice

**Date:** 2026-07-01 (Wednesday)  
**Topic Studied:** Why Monitoring Matters in DevOps  

##  Key Points Learned
1. **Detect Problems Early**  
   - Example: Disk usage reaches 90% → Alert generated before full.  

2. **Reduce Downtime**  
   - Example: Server crash on a streaming platform → Monitoring alerts engineers instantly.  

3. **Improve User Experience**  
   - Tracks response time, error rates, application speed.  
   - Example: Response time degradation from 500ms → 4s detected.  

4. **Supports Continuous Delivery**  
   - Monitoring verifies deployments.  
   - Example: Version 1.1 shows increased error rate → Bad deployment identified.  

5. **Capacity Planning**  
   - Monitoring helps plan upgrades before overload.  
   - Example: CPU growth 40% → 55% → 70% → 85% → Expansion planned proactively.
